# Branching Model of JuliaGenie

## Overview
This project follows a simplified GitFlow model.

## Branches
*   `master`: Production-ready code.
*   `develop`: Integration branch (optional, currently using trunk-based for initial setup).
*   `feature/*`: New features.
*   `fix/*`: Bug fixes.
*   `hotfix/*`: Critical production fixes.

## Workflow
1.  Create a branch from `master` (or `develop`).
2.  Implement changes.
3.  Open a Pull Request.
4.  Ensure CI passes.
5.  Merge.

## Work-In-Progress (WIP) Guidelines

*   **Limit:** 2 tasks per person.
*   **Focus:** Finish what you started before picking up new work.
*   **Stuck?** Mark as blocked and ask for help.
