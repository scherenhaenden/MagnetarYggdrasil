# Testing Strategy for Unison

## Types of Tests
*   **Unit Tests:** Verify individual functions and types in isolation.
*   **Integration Tests:** Verify interactions between layers (e.g., Service -> Repository).
*   **End-to-End Tests:** Verify the full API flow from request to response.

## Code Coverage
*   Target: 100% coverage.

## Bug Reporting Process
*   Report bugs in `BLOCKERS.md` or as GitHub issues.
*   Include reproduction steps and expected vs. actual behavior.
